配一下 mpv

创建于2023-08-08修改于2023-08-08访问量0标签mpv标签Arch Linux标签经验分享

mpv 是一款 神似 vim 的 媒体播放器，就是说第一眼有点劝退，配置项很多，但如果能花点时间进行配置还是挺香的，而且可以写脚本实现各种功能。

自从开始用 Linux 后我一直用的是 vlc，但总感觉有点捞（，最明显的是在暂停继续 / 快进后退时，经常会卡几秒，而且不是卡住不动而是边放边卡，感觉非常神秘，不知道能不能修，反正我之前简单搜过没修好。而 mpv 默认就没有这样的问题，早就想用了，但之前被它简陋的 UI 以及复杂的配置劝退了，拖到现在才入坑。

Warning

我是 Arch Linux + i3 + nnn 用户，下面有一些配置可能不适用于其他用户。

配置文件

  • ~/.config/mpv/mpv.conf: 主配置文件
  • ~/.config/mpv/input.conf: key binding 配置
  • ~/.config/mpv/scripts/: 脚本，放里面就会运行
  • ~/.config/mpv/script-opts/: 脚本的配置，例如 ~/.config/mpv/script-opts/osc.conf
  • /usr/share/mpv/scripts/: 一些自带的脚本，可以 symlink 到 ~/.config/mpv/scripts/

有个冷知识 (?) 可以注意一下：

Every flag option has a no-flag counterpart, e.g. the opposite of the --fs option is --no-fs. --fs=yes is same as --fs, --fs=no is the same as --no-fs.

输出质量

INI

profile=gpu-hq
profile=gpu-hq

High quality configurations - mpv - ArchWiki 给出了一些选项，其中 profile=gpu-hq 一般都是要加的。

scale 和 cscale 在片源分辨率不低于显示器分辨率时影响几乎没有，片源分辨率低时影响也不大，没必要加。主要是加了之后在 gpu-next 下不知道为什么有时候会黑屏，而且 ewa_lanczossharp 在 gpu-next 中 deprecated 了，需要用 ewa_lanczos 然后调 scale-blur。

后面的 interpolation 感觉也没必要，可能造成 “soap opera effect”，而且万一性能没跟上掉帧之类的就更惨了。

感觉 subtitle / audio track 只能 cycle 非常不好用；添加字幕几乎只能 drag and drop，对于 i3 + nnn 用户来说虽然有 dragdrop 插件 但也挺麻烦的。于是写了两个脚本（需要安装 kdialog）。

添加 track 文件

Lua
local mp = require 'mp'
local utils = require 'mp.utils'

local add_track = function(type, filter)
  local cwd = mp.get_property('working-directory')
  local parent_dir = utils.split_path(cwd)

  local dialog_result = mp.command_native {
    name = 'subprocess',
    args = {
      'kdialog',
      '--title',
      'Add ' .. type .. ' - mpv',
      '--getopenfilename',
      type == 'sub' and parent_dir or cwd,
      filter,
    },
    capture_stdout = true,
  }

  if dialog_result.status == 0 and dialog_result.stdout ~= nil then
    local path = dialog_result.stdout:gsub('\n', '')
    local add_result = mp.commandv(type .. '-add', path)
    if add_result == true then
      mp.osd_message(type .. ' added: ' .. path)
      return
    end
  end

  mp.osd_message('Failed to add ' .. type)
end

选择已有 track

Lua
local mp = require 'mp'

local select_track = function(type)
  local args = {
    'kdialog',
    '--title',
    'Select ' .. type .. ' track - mpv',
    '--geometry',
    '1280x720',
    '--radiolist',
    'Select ' .. type .. ' track',
    'no',
    '(no ' .. type .. ')',
    'on', -- the last one takes effect when there are multiple on
  }

  local count = mp.get_property_number('track-list/count')
  for i = 0, count - 1 do
    local track = mp.get_property_native('track-list/' .. i)
    if track.type == type then
      table.insert(args, tostring(track.id))
      table.insert(
        args,
        (track.title or track.lang or tostring(track.id))
        .. (type == 'audio' and (' (' .. track['demux-channel-count'] .. ' channels)') or '')
      )
      table.insert(args, track.selected and 'on' or 'off')
    end
  end

  local dialog_result = mp.command_native {
    name = 'subprocess',
    args = args,
    capture_stdout = true,
  }

  if dialog_result.status == 0 and dialog_result.stdout ~= nil then
    local sel = dialog_result.stdout:gsub('\n', '')
    local set_result = mp.commandv('set', type, sel)
    if set_result == true then
      local current = mp.get_property_native('current-tracks/' .. type)
      mp.osd_message(type .. ' chosen: ' .. (current == nil and '(none)' or current.title or current.lang or current.id))
      return
    end
  end

  mp.osd_message('Failed to select ' .. type)
end

mp.add_key_binding(nil, 'select-audio',    function() select_track('audio') end)
mp.add_key_binding(nil, 'select-video',    function() select_track('video') end)
mp.add_key_binding(nil, 'select-subtitle', function() select_track('sub')   end)

暂停图标

试过在暂停的时候显示一个图标，后来还是觉得去掉好一点，改成了 osd-msg cycle pause。

想要的话可以参考 @CogentRedTester - OSD-info when video is paused。需要注意的是这里用到了 “⏸”，而 color emoji 会显示为乱码，只能用 monochrome emoji，可以装一个 Noto Emoji Monochrome 加在 fontconfig 里。
