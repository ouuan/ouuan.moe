于是看了下这个人的其他项目，他也不是不关注安全，star 最高的还是个“Fast Elliptic Curve Cryptography in plain javascript”，随便搜了下：

It's really sad that cryptography- and security-related packages are done in this way - no release notes, no severity assessment, NOTHING.2

I'm always trying to keep security in mind, but it doesn't look like there is a way to mitigate timing attacks in this library without sacrificing the performance. JS implementation is about 10x-20x times slower than C already, and I don't want to make it even slower than this.3

我能说什么呢，可能这个人写的东西真的不太适合 be used for security purposes 吧（；但他是 software engineer at Signal，好可怕（（
